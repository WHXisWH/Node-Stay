## 2. 一言ナラティブ（叙述）
### 日本語
NodeStayは、ネットカフェの機器を収益権、計算資源利用権、施設利用権の3つの取引可能な経済的権利に分解する。

### English
NodeStay turns internet cafe machines into tradable economic rights across revenue, compute, and usage markets.

## 3. 30秒バージョン
> 日本のネットカフェには既に大量の高性能な機器が配備されていますが、そのほとんどの時間は単一の消費モデルにしか使われていません。  
> NodeStayが実現したいのは、1台の機器を「施設利用権」「計算資源利用権」「収益権」という3つの取引可能な権利に分解することです。  
> 店舗側はこれらの権利を販売でき、ユーザーは投資、取引、または利用が可能です。そして、その所有権と決済は Polygon と JPYC を通じて実現されます。

## 4. 2分バージョン
### 4.1 課題
日本のネットカフェには膨大な機器資産がありますが、これらの資産は「インターネット利用料」という単一の収入源しか持っていません。  
日中のオフピーク時には利用率が低く、アイドル状態の機器に「第二の収入曲線」が形成されていません。また、外部の投資家がその収益に参加する手段もありません。

### 4.2 解決策
NodeStayは、1台の機器を以下の3つの経済的権利に分解します：
1. 収益権：投資家が将来の収益分配を購入できる  
2. 計算資源利用権：開発者が空き時間の計算リソースを購入できる  
3. 施設利用権：ユーザーが機器の利用時間を購入し、それを譲渡できる  

### 4.3 なぜWeb3なのか
これは単なる決済の問題ではなく、現実世界の機器資産の権利確定（確権）、取引、および自動分配の問題だからです。  
ブロックチェーンが「信頼できるアセットレイヤー」を担い、JPYC が「安定した決済レイヤー」を担います。

### 4.4 最終的なポジショニング
NodeStayは単なるネットカフェ向けソフトウェアではなく、以下を目指します：

> **現実世界の機器資産市場（Machine Asset Market）**

## 5. なぜネットカフェなのか（Q&A標準回答）
1. ネットカフェには既に大量の高性能機器が配備されているから  
2. それらの機器には明確なアイドル時間（空き時間）があるから  
3. 店舗自体に電力、ネットワーク、営業体制、およびタイムスライス（時間貸し）の仕組みが既に備わっているから  
4. 1台の機器が「リアルな消費者」と「計算タスク」の両方に同時に、あるいは切り替えてサービスを提供できるから  
5. したがって、ネットカフェは MachineFi を検証するのに最適な最初のシーンである

## 6. なぜWeb3でなければならないのか（Q&A標準回答）
NodeStayは単なる決済プロジェクトではなく、以下の機能を提供するからです：
- 機器資産の権利確定（確権）
- 権利の譲渡
- 自動決済
- 収益分配

ベストな表現：
> ブロックチェーンの役割は、機器に関する権利を、検証可能で譲渡可能、かつ自動分配可能な経済的資産へと変えることにあります。

## 7. なぜJPYCなのか（Q&A標準回答）
JPYCの価値は「独自の通貨を発行すること」ではなく、以下にあります：
1. 日本円建ての安定した決済  
2. 日本国内の価格表示に適している  
3. 3つの市場における統一された決済手段  
4. 自動分配、デポジット、および周期的な収益分配に最適である

## 8. なぜクレジットカードではいけないのか（Q&A標準回答）
クレジットカードは従来の小売決済には適していますが、NodeStayには以下の機能が必要です：
- 高頻度・少額・動的な決済
- プログラム可能な資産取引
- 自動的な収益分配（分帳）
- 周期的な収益分配ロジック

ベストな表現：
> Credit cards are good for retail payments.  
> NodeStay needs a programmable economic system.

## 9. なぜ AWS / GCP ではないのか（Q&A標準回答）
NodeStayはクラウドを代替しようとしているのではなく、以下を活性化しようとしています：
- 既に存在している現実世界の機器ネットワーク
- オフピーク時のアイドルリソース
- エッジコンピューティングや短時間のタスクシーン

ベストな表現：
> AWS builds new compute infrastructure.  
> NodeStay activates idle compute infrastructure that already exists.

## 10. プロジェクト最大のイノベーションポイント
### 10.1 表面的なイノベーション
3つの市場モデル：
- Revenue Market（収益市場）
- Compute Market（計算資源市場）
- Usage Market（施設利用市場）

### 10.2 本質的なイノベーション（最重要）
真の革新は「3つの市場があること」ではなく、以下にあります：

> **1台の物理的な機器を多層的な経済的権利（Economic Rights Stack）に分解したこと**

```text
Machine
├─ Revenue Right（収益権）
├─ Compute Right（計算資源利用権）
└─ Usage Right（施設利用権）
```

## 11. MVPの定義（ピッチ・デモ用）
### 絶対に譲れないコアMVP
1. 機器のオンチェーン登録  
2. Usage Right の発行  
3. JPYC による支払い  
4. オンチェーン取引記録の表示  
5. check-in / check-out のオフチェーンクローズドループ

### デモで見せるべき内容
- 機器登録の成功
- UsageRight の正常な mint
- ユーザーによる購入の成功
- オンチェーン取引履歴の確認
- 利用権のステータス変化（未使用 → 使用済み）

### 現時点では必須ではないもの
- 完全な GPU 市場
- 完全な収益権市場
- 複雑な動的価格設定
- 投資家向けフルモジュール

## 12. GTM（Go-To-Market）戦略の提案
### 第一段階

- ハイスペック機器による少規模な実証実験
- まずは Usage 市場を構築
- 実際のオンチェーン決済により信頼性を高める

### 第二段階
以下の導入：
- 複数店舗共通の Usage 市場

### 第三段階
以下の導入：

- アイドル機器の計算リソーススケジューリング

## 14. 想定される厳しい質問と回答（Q&A）
### Q1：ネットカフェにNFTを追加しただけではないですか？
A：違います。核心はNFTではなく、機器を多層的な経済的権利に分解し、その取引と分配のルールを定義したことにあります。

### Q2：ブロックチェーンを使わなくても実現できるのでは？
A：一部の機能は可能ですが、検証可能な権利確定、透明性の高い権利譲渡、および自動的な収益分配を、現在と同じ方法で実現することは不可能です。

### Q3：なぜ収益権を買う人がいるのですか？
A：現実の機器が生み出す Usage / Compute の実収益と紐付いているからです。純粋な仮想の物語ではありません。

### Q4：なぜ店主がこれに協力するのですか？
A：機器を単一の収入源から3層の収入源に変え、将来の収益の一部を早期に現金化できるからです。

### Q5：なぜユーザーが施設利用権を買うのですか？
A：透明性が高く、譲渡が可能であり、将来的には複数店舗で共通して使える「時間の資産」になるからです。

### Q6：GPU市場は難易度が高すぎませんか？
A：はい、非常に高いです。そのため MVP では完全な GPU 市場を第一目標にせず、まずは Usage のトークン化と JPYC 決済の検証を優先します。

### Q7：Airbnb や REIT との関係は何ですか？
A：Airbnb はアイドルスペースを活性化し、REIT は不動産収益を金融化しました。NodeStay は「現実世界の機器資産」を金融化します。

### Q8：プロジェクトの最終的なゴールは何ですか？
A：単一のネットカフェ製品ではなく、Machine Asset Network（機器資産ネットワーク）を構築することです。

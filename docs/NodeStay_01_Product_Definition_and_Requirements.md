# NodeStay / MachineFi プロダクト定義および要件定義書
バージョン：v1.0 Draft  
ポジショニング：Polygon + JPYC ベースの機器資産3市場プロトコルおよびアプリケーション層プロダクト

## 1. ドキュメントの目的
本ドキュメントは、NodeStayプロジェクトのプロダクト定義、システム境界、ロールモデル、コアな権利設計、およびMVP要件を統一することを目的とする。  
NodeStayの最初のユースケースは日本のネットカフェであるが、長期的なポジショニングは単なる「ネットカフェ向けソフトウェア」ではなく、以下を目指すものである：

> **MachineFi：現実世界の機器資産における金融化および市場化のインフラストラクチャ。**

## 2. プロジェクトの一言説明
**NodeStayは、ネットカフェの機器を収益権、計算資源利用権、施設利用権の3つのプログラム可能な経済的権利に分解し、ユーザーがこれらの権利を投資、取引、または利用することを可能にする。**

## 3. なぜネットカフェなのか
### 3.1 ネットカフェは天然の現実資産（RWA）ノードである
1. 既に大量の高性能な機器が配備されている  
2. 安定した電力、ネットワーク、営業場所、および席管理が備わっている  
3. ピーク時とオフピーク時が明確であり、大量のアイドル時間（空き時間）が存在する  
4. 機器は「人」へのサービス提供と、「計算タスク」へのリソース提供の両方が可能である  
5. タイムスライス（時間貸し）のビジネスモデルが天然の形で備わっている

### 3.2 従来モデルの課題
- 機器資産が単一の収入源しか生み出せない
- 日中や低稼働時間帯の利用率が低い
- 収益と資産の間に取引可能なレイヤーが欠如している
- ユーザーの時間権益を標準化して流通させることができない
- 投資家が個別の機器収益に直接参加することができない

### 3.3 コアとなる判断
> **日本には既に分散型の機器ネットワークが存在しているが、まだ金融化されていないだけである。**

## 4. プロダクトの核心：機器資産3市場モデル
基盤となる資産は **Machine Asset（機器資産）** である。  
各機器は以下の3つの経済的権利に分解される：

```text
Machine Asset
├─ Revenue Right   （収益権）
├─ Compute Right   （計算資源利用権）
└─ Usage Right     （施設利用権）
```

## 5. 3つの経済的権利の定義
### 5.1 Revenue Right（収益権）
特定の機器の将来の収益の一部の分配を受ける権利を表す。
- 例：Machine #12 の今後12ヶ月間における収益の20%を受け取る権利
- 収益源：Usage（施設利用） + Compute（計算資源利用）
- 対象：投資家、二次市場トレーダー、経営者の資金調達

### 5.2 Compute Right（計算資源利用権）
特定の機器の特定の時間枠における計算資源の占有権を表す。
- 例：Machine #27 における 14:00–18:00 のGPU使用権
- 対象：AI推論、レンダリング、ZK（ゼロ知識証明）タスクの需要家

### 5.3 Usage Right（施設利用権）
特定の機器または機器プールの特定の時間枠における物理的な利用権を表す。
- 例：3時間パック、深夜8時間パック、指定席の時間枠

## 6. ユーザーロール体系
### 6.1 ネットカフェオーナー / 機器運営者
1. 機器資産のオンチェーン登録  
2. 機器の将来の収益権のオンチェーン販売  
3. アイドル状態の計算資源利用権のオンチェーン販売  
4. 機器の時間利用権（施設利用権）のオンチェーン販売  

### 6.2 利用者（User）
Usage Rightを購入し、店舗で実際に機器を利用する。

### 6.3 計算資源の買い手（Compute Buyer）
Compute Rightを購入し、AI、レンダリング、ZK等のタスクを送信する。

### 6.4 投資家（Investor）
Revenue Rightを購入し、将来の収益分配を受け取る。

### 6.5 トレーダー（Trader）
二次市場において Usage / Compute / Revenue の3つの権利を取引する。

## 7. なぜWeb3が必須なのか
NodeStayは単なる決済プロジェクトではなく、以下の機能を提供する：
- 機器資産の権利確定（確権）
- 権利の譲渡
- 自動決済
- 収益分配

プロジェクトにおけるブロックチェーンの役割は、機器に関する権利を、検証可能で譲渡可能、かつ自動分配可能な経済的資産へと変えることにある。

## 8. なぜJPYCを使用するのか
JPYCの主要な役割は **安定した決済資産（ステーブルコイン）** である。
以下の用途で使用される：
- Usage Right の支払い
- Compute Right の支払い
- デポジットの凍結および解放
- Revenue Right の周期的な収益分配
- プラットフォームと加盟店間の決済

## 9. プロダクトの境界
### 本プロジェクトが現時点で「ない」もの
- 純粋なGPUクラウドベンダーではない
- ホテル予約プラットフォームではない
- 一般的なチケットシステムではない
- 単一のNFTコレクションプロジェクトではない

### 本プロジェクトの本質
> **現実世界の機器資産市場（Machine Asset Market）**

## 10. MVPの範囲（推奨）
### Phase 1
1. Machine Registry（機器登録）  
2. Usage Right のオンチェーン発行および譲渡  
3. JPYC決済およびオンチェーン記録  
4. オフチェーンでの check-in / check-out  
5. シンプルな二次取引

### Phase 2
1. Compute Right  
2. シンプルな計算タスクのスケジューリング  
3. 中断可能な決済ロジック

### Phase 3
1. Revenue Right  
2. 周期的な収益分配  
3. 投資家向け市場

## 11. 成功の基準
1. 機器が多層的な収入を生み出せるか  
2. 権利が実際に取引されるか  
3. 投資家が検証可能な収益を得られるか  
4. 運営者が導入を希望するか  
5. ユーザーが購入 / 利用 / 転売を希望するか

## 12. 最終定義
### 日本語
NodeStayは、ネットカフェの機器を収益権、計算資源利用権、施設利用権に分解する、現実世界の機器資産市場である。

### English
NodeStay is a machine-based real-world asset marketplace that splits internet cafe machines into revenue, compute, and usage rights.
